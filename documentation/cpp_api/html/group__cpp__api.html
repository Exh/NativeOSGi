#define OSGI_DECLARE_SERVICE_INTERFACE (   IFace,
  Identifier 
)

This macro associates the given Identifier (a string literal) to the interface class called IFace.

The Identifier must be unique. For example:

#include <osgi/MacroUtils.h>
#define ICameraDevice_ID com_mycompany_ICameraDevice_1_0
#define ICameraDevice_NAME OSGI_XSTR(ICameraDevice_ID)
struct ICameraDevice { ... };
OSGI_DECLARE_SERVICE_INTERFACE(ICameraDevice, ICameraDevice_NAME)

This macro is normally used right after the class definition for IFace, in a header file.

If you want to use OSGI_DECLARE_SERVICE_INTERFACE with interface classes declared in a namespace then you have to make sure the OSGI_DECLARE_SERVICE_INTERFACE is not inside a namespace though. For example:

#include <osgi/MacroUtils.h>
#define MyInterface_ID com_mycompany_MyInterface_1_0
#define MyInterface_NAME OSGI_XSTR(MyInterface_ID)
namespace Foo
{
struct MyInterface { ... };
}
OSGI_DECLARE_SERVICE_INTERFACE(Foo::MyInterface, MyInterface_NAME)

Typedef Documentation

typedef OSGI_UNORDERED_MAP_TYPE<ci_string, Any> osgi::ServiceProperties

A hash table based map class with case-insensitive keys.

This class uses ci_string as key type and Any as values. Due to the conversion capabilities of ci_string, std::string objects can be used transparantly to insert or retrieve key-value pairs.

Note that the case of the keys will be preserved.

template<typename ValueType >
ValueType osgi::any_cast ( const Any &  operand)

any_cast operator used to extract a copy of the ValueType from an const Any&.

Example Usage:

MyType tmp = any_cast<MyType>(anAny)

Will throw a BadCastException if the cast fails. Dont use an any_cast in combination with references, i.e. MyType& tmp = ... or const MyType& = ... Some compilers will accept this code although a copy is returned. Use the ref_any_cast in these cases.
